# mbrace K/B ファイル形式メモ

検分対象: data/raw/k250705.lzh, b250705.lzh（戸田セクション含む実データ）ほか。
文字コード: Shift-JIS固定長テキスト（`iconv -f SHIFT-JIS -t UTF-8` で変換後も1文字=1論理カラムの対応関係を維持することを確認済み）。

## 共通構造

- ファイル全体: `STARTK`/`STARTB` … `FINALK`/`FINALB`
- 場ごとのセクション: `NNKBGN` … `NNKEND`（Kファイル）、`NNBBGN` … `NNBEND`（Bファイル）
  - NN = 2桁の場コード。戸田 = `02`
  - 場コードは標準の全国ボートレース場コードに一致することを確認（例: 24=大村, 23=唐津, 22=福岡, 18=徳山, 16=児島, 15=丸亀, 14=鳴門, 10=三国, 09=津, 08=常滑, 02=戸田）
- レースヘッダ行: `  NR       タイトル                 H距離m  天候  風  風向　 風速m  波　  波cmm`
  - 例: `   1R       予選　　　　                 H1800m  曇り  風  無風　 0m  波　  0cm`
  - 正規表現: `/^\s*(\d{1,2})R\s+(.+?)\s+H(\d+)m\s+(\S+)\s+風\s+(\S+)\s+(\d+)m\s+波\s+(\d+)cm/`

## Kファイル（競走成績）

### 決まり手
着順表ヘッダ行の末尾に決まり手（逃げ／差し／まくり／まくり差し／抜き 等）が付記される。
例: `  着 艇 登番 　選　手　名　　ﾓｰﾀｰ ﾎﾞｰﾄ 展示 進入 ｽﾀｰﾄﾀｲﾐﾝｸ ﾚｰｽﾀｲﾑ 差し　　　`
→ `ﾚｰｽﾀｲﾑ` の後の文字列が decision。

### 着順行（固定カラム、1行58文字）
文字インデックス（0始まり、`slice(a,b)`）で確認済み:

| フィールド | 範囲 | 内容 | 備考 |
|---|---|---|---|
| finish | [0,4) | 着順 `"01"`〜`"06"` or `"F"`,`"K0"`,`"S0"`,`"S2"` 等 | フライング=F、欠場=K0、失格/スタート事故=S+数字 |
| boat | [4,7) | 艇番 1〜6 | |
| racer | [7,13) | 登録番号（4桁） | |
| name | [13,21) | 選手名（8文字固定、全角スペース区切り含む） | 全角スペース除去して整形 |
| motor | [21,25) | モーター番号（当日） | 欠場時は `"K ."` |
| boat_id | [25,29) | ボート番号（当日） | |
| exhibition | [29,36) | 展示タイム | 欠場時 `"K ."` |
| course | [36,39) | 進入コース | 欠場時は空白 |
| st | [39,47) | スタートタイミング | フライング時 `"F0.02"` 等 |
| racetime | [47,58) | レースタイム | 未着（流し等）は `". ."` |

異常系finishコードは文字列のまま `finish` に格納し、レコードは除外しない（欠場等でも艇のエントリ行として残す）。

### 払戻行
`単勝`/`複勝`/`２連単`/`２連複`/`拡連複`/`３連単`/`３連複` のラベル＋値。空白区切りでトークン化してパース。
必要な5種のみ抽出: `単勝`(win) `２連単`(exacta) `２連複`(quinella) `３連単`(trifecta) `３連複`(trio)。
`複勝`・`拡連複`は本スキーマ対象外のためスキップ。

特殊系:
- `特払い`（例: `単勝        特払い   70`）→ comb:null, amount:金額, special:"tokubarai"
- `不成立`（未確認、想定形式similar）→ comb:null, amount:null, special:"fusei_ritsu"

## Bファイル（番組表）

### 未解決事項（正直な報告）
- **f_count（F数）/ l_count（L数）/ avg_st（平均ST）はBファイル・Kファイルどちらにも該当ラベル付きフィールドが存在しないことを確認した**（`grep -n "Ｆ\|平均\|ＳＴ\|F数\|L数"` で検索したが該当なし）。
  - Bファイル末尾の「早見」列（1〜2桁の数値）が該当情報を符号化している可能性はあるが、ラベルが無く意味を断定できないため採用しない。
  - よって出力JSONの `f_count` / `l_count` / `avg_st` は **すべて `null`** とし、データソースに存在しないことを明記する。将来的に別ファイル形式（拡張番組表等）が入手できれば埋める。
- `不成立` の実際の行フォーマットは今回の収集データ中に出現しなかったため未確認。パーサーは `特払い` と同様のトークン構造を仮定して防御的に実装した（実データで出現した場合は docs/parse_errors.log に記録されるよう設計）。

## 結合ロジック
K・Bとも `NNKBGN/NNBBGN` セクション内のレース番号・艇番をキーに結合。
- K由来: finish, exhibition, course, st, motor(当日), boat_id(当日), racetime, 払戻
- B由来: class, age, branch, nat_win, nat_2r, loc_win, loc_2r, motor_2r, boat_2r
- 艇番が一致しない（差し替え等でBに該当エントリが無い）場合はB由来フィールドをnullとし、docs/parse_errors.logには**記録しない**（軽微な欠損として許容、レース自体は出力する）。

### データなし判定（非開催日・rno範囲外等）
`<span class="heading1_mainLabel">データがありません。</span>` が出現したら `noData: true` を返す。

### 選手情報テーブル（`<div class="table1"><table class="is-w748">`）
6艇分、各艇 `<tbody class="is-fs12">` で区切られる（1つ目は前にヘッダ`<thead>`）。
各tbody内は4つの`<tr>`（rowspan混在）:
- tr1: 枠番号(`is-boatColor{n}`のtd) / 写真 / 選手名 / **体重**(`rowspan="2"`, 例`"52.4kg"`) / **展示タイム**(`rowspan="4"`) / **チルト**(`rowspan="4"`) / **プロペラ**(`rowspan="4"`) / **部品交換**(`rowspan="4"`, `<ul class="labelGroup1"><li><span class="label4...">ラベル</span></li>...</ul>`) / "R"ラベル / 前走レース番号値
- tr2: "進入"ラベル / 前走進入コース値
- tr3: **調整重量**(`rowspan="2"`, 体重欄の2行目) / "ST"ラベル / 前走ST値
- tr4: "着順"ラベル / 前走着順値

**重要な訂正**: tr1〜tr4の末尾2列（"R"/"進入"/"ST"/"着順"のラベル付き値）は見出し的には「前走成績」列（当該開催内の直前レースの成績）であり、**今回レースの進入予想・スタート展示ではない**。今回レースの進入予想・ST予想は別テーブル（下記）。
未入力セルは `&nbsp;` がそのまま入る（数値ではなく文字列`&nbsp;`）。パーサーは `&nbsp;` を `null` に変換する。

体重・調整重量は常時取得可能（実測値あり）。展示タイム・チルト・プロペラ・部品交換は**展示走行後でないと空欄**（今回2026-07-12 R1/R2はどちらも展示前の時間帯にfetchしたため、いずれも空欄`&nbsp;`だった＝構造は確認できたが実数値の取得は今回**未確認**）。

### スタート展示テーブル（`<table class="is-w238">`、見出し「スタート展示」）
`<thead>`列: コース／並び／ST。`<tbody class="is-p10-0">` に6行。
展示前は各行 `<td colspan="3"></td>` の空セル（2026-07-12 R1/R2で確認）。
**展示後の実際のセル構造（コース/並び/STがそれぞれ独立tdになるか等）は未確認**。パーサーは colspan="3" の空行はnullとして扱い、それ以外の行は3つのtd値を順に「course/formation/st」として仮読みする防御的実装とし、想定外の形が来ても例外を投げず該当艇をnullにする設計にした。**実際に展示後のページで再検証するまでは、この部分の出力は「取れない可能性がある」として扱うこと。**

### 水面気象情報（`<div class="weather1">`）
常時取得可能（展示前後を問わない）。
- 気温: `.is-direction .weather1_bodyUnitLabelData` 内、例 `"28.0℃"`
- 天候ラベル: `.is-weather .weather1_bodyUnitLabelTitle` 内、例 `"曇り"`
- 風速: `.is-wind .weather1_bodyUnitLabelData` 内、例 `"0m"`
- 風向: `.is-windDirection` 内 `<p class="weather1_bodyUnitImage is-wind{NN}">` の**アイコンclass番号のみ**。テキストの方角ラベルはページ内に存在しない。NN=1〜16が16方位、17を無風時に確認（2026-07-12は風速0mでis-wind17）だが、**NN→方角名の対応表は未確認・未実装**。出力は`windDirectionIconNumber`（数値）のみとし、方角名は`null`のまま返す（正直な報告：方角名は取れない）。
- 水温: `.is-waterTemperature .weather1_bodyUnitLabelData` 内、例 `"28.0℃"`
- 波高: `.is-wave .weather1_bodyUnitLabelData` 内、例 `"0cm"`
